Agent stack

Not a reminder bot. A bounded business operator.

The contest asks for agents that run real operations. SOLVENT shows the whole operation, including the safety boundary that keeps it viable.

01 / Stripe

Reads the business state

Amount, age, failed payment reason, customer history, and hosted payment links become the working context for the agent.

02 / Hermes

Chooses the recovery move

Hermes drafts the gentlest message likely to work. The product can raise the floor, but the model cannot weaken policy.

03 / NVIDIA

Screens before execution

Nemotron Content Safety checks every outgoing message. Unsafe text routes to review instead of reaching customers.

Demo storyboard

The 90-second story is visible on the page.

This is designed to read cleanly in a hackathon video: one invoice, one agent loop, one safety gate, one Stripe settlement path.

00:00

Invoice stalls

A customer is 17 days overdue. The agent receives Stripe context instead of a vague prompt.

Hermes drafts

The draft is checked for pressure, safety, and tone before it can leave the system.
